# 셰이더 설정

## lilToon

모든 파츠에 **lilToon**을 사용합니다. 새 머티리얼을 생성할 때는 이 페이지의 값에 맞춰 주십시오.

### 라이팅 설정（모든 파츠 공통값）

`Brightness` / `Lighting(mono)` 파라미터와 연동됩니다. 값이 다르면 밝기를 조정할 때 새 파츠만 동작이 달라집니다.

| 프로퍼티 | 초기값（라디얼 0） | 최댓값（라디얼 1） |
|-----------|------|------|
| `_LightMinLimit` | 0.05 | 1.0 |
| `_MonochromeLighting` | 0.5 | 1.0 |
| `_LightMaxLimit` | 1.0 | — |
| `_lilDirectionalLightStrength` | 1.0 | — |
| `_VertexLightStrength` | 0 | — |

### 스텐실 시스템

스텐실은 렌더링 시 버퍼의 수치를 이용해 픽셀을 통과시킬지 거부할지를 제어하는 구조입니다. lilToon에서는 다음 두 가지 프로퍼티를 사용합니다.

| 프로퍼티 | 역할 |
|-----------|------|
| **StencilRef** | 비교에 사용하는 참조값（0〜255） |
| **StencilComp** | 비교 방법. `Always`：항상 통과시킴（버퍼에 값을 씀）. `Equal`：버퍼 값이 Ref와 일치하는 픽셀만 렌더링. `NotEqual`：버퍼 값이 Ref와 일치하지 않는 픽셀만 렌더링. |

3+1에서는 **Ref=1（앞머리 컷아웃용）**과 **Ref=2（만화 효과 마스크용）**, 두 가지 채널을 사용합니다.

#### 앞머리 컷아웃（눈썹 · 아이라인）

앞머리가 눈썹이나 아이라인 위에 겹쳐도 눈썹이 투과되어 보이도록 하는 시스템입니다.

1. 얼굴 메시（SA_BODY_B / SA_BODY_C）가 `StencilComp: Always`로 렌더링되어, 얼굴이 표시되는 픽셀의 스텐실 버퍼에 **1**을 씁니다.
2. 앞머리 메시（SA_HAIR_B）는 `StencilComp: NotEqual / StencilRef: 1`로 렌더링되므로, 버퍼에 1이 쓰인 픽셀（= 얼굴 위）에서는 렌더링을 건너뜁니다.
3. 결과적으로 앞머리가 얼굴과 겹치는 부분만 잘려나가 눈썹과 아이라인이 보입니다.

| 머티리얼 | 역할 | StencilRef | StencilComp |
|-----------|------|-----------|-------------|
| SA_BODY_B / SA_BODY_C | 얼굴（쓰기） | 1 | Always |
| SA_HAIR_B | 앞머리（얼굴 위를 컷아웃） | 1 | NotEqual |

**새 앞머리를 추가할 경우：** 머티리얼에 `StencilComp: NotEqual / StencilRef: 1`을 설정해 주십시오. 설정하지 않으면 눈썹과 아이라인이 앞머리에 가려집니다.

#### 만화 효과 마스크

창백함 등의 얼굴 효과를 얼굴 피부 영역에만 렌더링하는 시스템입니다.

1. 얼굴 피부 메시（SA_BODY_A）가 `StencilComp: Always`로 렌더링되어, 피부가 표시되는 픽셀의 스텐실 버퍼에 **2**를 씁니다.
2. 만화 효과（SA_EFFECT_A）는 `StencilComp: Equal / StencilRef: 2`로 렌더링되므로, 버퍼에 2가 쓰인 픽셀（= 피부 위）에만 표시됩니다.
3. 결과적으로 효과가 얼굴 윤곽을 따라 자연스럽게 마스킹됩니다.

| 머티리얼 | 역할 | StencilRef | StencilComp |
|-----------|------|-----------|-------------|
| SA_BODY_A | 얼굴 피부（쓰기） | 2 | Always |
| SA_EFFECT_A | 만화 효과（피부 위에만 렌더링） | 2 | Equal |

**얼굴에 겹쳐 표시하는 효과를 추가할 경우：** `StencilComp: Equal / StencilRef: 2`를 설정하면 피부 영역에만 한정해 렌더링할 수 있습니다.

#### 새 파츠 추가 시 체크리스트

| 추가할 파츠 | 필요한 스텐실 설정 |
|---------------|-----------|
| 앞머리 · 앞머리와 겹치는 파츠 | `StencilComp: NotEqual / StencilRef: 1` |
| 얼굴에 겹쳐 표시하는 효과 | `StencilComp: Equal / StencilRef: 2` |
| 일반 의상 · 액세서리 | 불필요（기본값 그대로） |

### 렌더 큐

| 머티리얼 | RenderQueue | 비고 |
|-----------|-------------|------|
| SA_BODY_A | 2000 | 스텐실 쓰기 |
| SA_EFFECT_A | 2460 | 만화 효과 |
| SA_HAIR_A | 2500 | 머리카락 |

새 파츠의 배치 기준：
- 일반 의상 · 액세서리：2000 전후
- 투명 · 반투명 효과：2450〜2490
- 머리카락 위에 렌더링할 것：2500 이상

## HINASAKI SFX SHADER

땀, 눈물, 모자이크 등의 효과에는 lilToon이 아닌 **HINASAKI SFX SHADER**를 사용합니다. GrabPass를 통한 배경 굴절과 OpenLit 라이팅을 갖춘 커스텀 셰이더 모음으로, 머티리얼의 '효과 모드' 드롭다운에서 셰이더를 전환할 수 있습니다.

| 셰이더 | 용도 |
|-----------|------|
| **Teardrop**（うるうる目・落涙） | 눈의 촉촉함, 눈물, 흘러내리는 물방울 |
| **Waterdrop**（水滴） | 땀 등 표면 전체에 흐르는 물방울 |
| **Mosaic**（平面モザイク） | 배경의 모자이크 처리 |

세 가지 효과 레이어로 구성되어 있습니다.

#### 물방울 텍스처（떨어지는 물방울）

| 프로퍼티 | 설명 |
|-----------|------|
| 물방울 텍스처 / Drop Color | 떨어지는 물방울의 형상 텍스처（흰색 실루엣）와 하이라이트 색상 |
| 페이드 마스크 | 알파 채널로 물방울이 사라지는 페이드를 제어하는 마스크 |
| 가로 크기 / 세로 크기 | 텍스처 표시 크기（1=기본, 중심 위치는 변하지 않음） |
| 물방울 불투명도 | 효과 전체의 알파. 애니메이션으로 페이드인/아웃 가능 |

#### 물방울 설정（녹색 선）

| 프로퍼티 | 설명 |
|-----------|------|
| X 위치 | 물방울의 가로 기준 위치. 디버그 모드 녹색 선을 확인하며 조정 |
| 가로 랜덤 폭 | X 좌표의 랜덤 산포 폭. 값이 클수록 좌우로 넓게 흩어짐 |
| 흔들림 주파수 | 낙하 중 좌우 흔들림 주파수. 값이 클수록 세밀하게 흔들림 |
| 흔들림 폭 | 낙하 중 좌우 흔들림 폭. 값이 클수록 크게 흔들림 |

#### 눈물 텍스처（눈 안쪽에 고이는 눈물）

| 프로퍼티 | 설명 |
|-----------|------|
| 눈물 텍스처 / Tear Color | 눈 안쪽에 고이는 눈물의 형상 텍스처와 하이라이트 색상 |
| 노이즈 텍스처 | 굴절 노이즈와 촉촉한 효과에 공용으로 사용하는 노이즈 |
| 가로 크기 / 세로 크기 | 텍스처 표시 크기 |
| 눈물 불투명도 | 애니메이션으로 페이드인/아웃 가능 |

#### 눈물 설정（노란색 선）

| 프로퍼티 | 설명 |
|-----------|------|
| X 위치 | 눈물의 가로 위치. 디버그 모드 노란색 선으로 확인 |
| Y 위치 | 눈물의 세로 위치. 촉촉한 효과의 기준 Y값도 됨 |
| 노이즈 크기 | 굴절 노이즈의 UV 스케일. 값이 클수록 입자가 细밀해짐 |
| 노이즈 속도 | 노이즈 스크롤 속도. 0.1〜0.3 정도를 권장 |
| 노이즈 강도 | 굴절 노이즈의 강도. 너무 크게 하면 부자연스러워짐 |

#### 촉촉한 텍스처 및 설정（파란색 선）

| 프로퍼티 | 설명 |
|-----------|------|
| 촉촉한 마스크 | 알파 채널로 촉촉한 효과의 적용 범위를 제어하는 마스크 |
| Y 위치（TearY에 가산） | 촉촉한 효과의 시작 Y 좌표. 눈물의 Y 위치에 가산하여 결정 |
| 노이즈 크기 / 속도 | 촉촉한 효과에 사용하는 노이즈의 UV 스케일과 스크롤 속도 |
| 촉촉한 강도 | 광택 효과의 강도. 0=무효 / 1.0=최대 |
| 노이즈 거리 보정（m） | 카메라 거리에 따른 노이즈 크기 자동 보정의 기준 거리 |
| 촉촉한 페이드 | 굴절 왜곡의 영향도. 애니메이션으로 페이드인/아웃 가능 |

#### 효과 전체 설정

| 프로퍼티 | 설명 |
|-----------|------|
| 텍스처 강도 | 물방울 · 눈물 하이라이트의 밝기. 0=비표시 / 1.0=최대 |
| 굴절 강도 | 배경의 굴절（왜곡）강도 |
| 낙하 속도 | 물방울 낙하 애니메이션 속도（0.01〜3） |

### Mosaic（HSFX_Mosaic）

<img src="/img/sfx-shader/mosaic-sample.webp" alt="모자이크 샘플" style={{borderRadius:'12px', width:'100%', aspectRatio:'4/3', objectFit:'cover'}} />

메시에 비치는 배경을 모자이크 처리하는 셰이더입니다. 프로퍼티는 하나뿐입니다.

<img src="/img/sfx-shader/mosaic.webp" alt="HSFX_Mosaic 인스펙터" style={{borderRadius:'8px', width:'100%'}} />

| 프로퍼티 | 설명 |
|-----------|------|
| 모자이크 강도（`_MosaicPercentage`） | 모자이크 블록의 크기. 0=모자이크 없음（선명） / 1=최대 모자이크 |

카메라와의 거리 및 시야각（FOV）에 따라 블록 크기가 자동으로 보정되므로 멀리서 보아도 균일한 모자이크로 보입니다. **카메라가 메시 밖에 있는 경우**에는 표면에, **안쪽에 있는 경우**에는 이면 전체에 모자이크가 적용됩니다.

### 디버그 모드（그리드 모드）

Teardrop · Waterdrop 공통. 위치 조정용 보조선을 표시합니다.

| 모드 | 표시 |
|--------|------|
| None | 일반 표시（기본값） |
| Transparent | 보조선만（배경 투과） |
| Black | 보조선만（검정 배경） |

보조선 색상：녹색=물방울 범위, 노란색=눈물 위치, 파란색=촉촉한 효과 경계

:::warning
None 이외의 상태로 업로드하면 게임 내에도 그리드가 표시됩니다. 업로드 전에 반드시 **None**으로 되돌려 주십시오.
:::

### OpenLit 라이팅 설정

Teardrop · Waterdrop은 lilToon과 동일한 OpenLit 라이브러리를 사용합니다. 새 머티리얼을 생성할 때는 본체와 동일한 값으로 맞춰 주십시오.

| 프로퍼티 | 값 | 설명 |
|-----------|-----|------|
| `_LightMinLimit` | 0.05 | 어두운 환경에서의 최저 휘도 |
| `_LightMaxLimit` | 1.0 | 밝은 환경에서의 상한 휘도 |
| `_MonochromeLighting` | 0 | 컬러 라이트를 그레이스케일로 적용（1.0=모노크롬） |
| `_VertexLightStrength` | 0 | 주변 포인트 라이트의 영향 강도 |
| `_AsUnlit` | 0 | 1.0으로 설정하면 라이팅을 무시하고 일정한 밝기로 표시 |
