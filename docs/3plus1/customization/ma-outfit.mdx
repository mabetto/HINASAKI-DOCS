# 髪・衣装の追加（Modular Avatar）

Modular Avatar（MA）による衣装の基本セットアップ（MA Merge Armature など）は[公式チュートリアル](https://modular-avatar.nadena.dev/ja/docs/tutorials)を参照してください。

このページでは、MAで衣装が動く状態になっている前提で、**３＋１のフォーマットに合わせる**手順を説明します。

<WarningCard>
**複数衣装を1アバターにまとめることは推奨しません。**
メニュー切り替えで複数の衣装を1アバターに詰め込むと、全衣装のテクスチャ・メッシュが常に同梱されるため、アバターのダウンロードサイズが数百 MB 規模になることがあります。衣装ごとに別のアバターとしてアップロードすることを検討してください。
</WarningCard>

<WarningCard>
**SA_COS_A は先に削除してください。**
新しい衣装を追加する前に、Hierarchy から `SA_COS_A` オブジェクトを削除します。SA_COS_A を残したまま別の衣装を追加すると、クローゼットメニューに COS_A のトグルが残り続け、パラメーターや見た目が意図しない状態になります。
</WarningCard>

---

## クローゼットメニューに追加する

３＋１のクローゼットは `3+1 OutfitMenu.asset` というサブメニューに各衣装がトグルを差し込む構造になっています。COS_A はこのサブメニューへ `SA_COS_A OutfitMenu.asset`（`CostumeHead` / `CostumeBody` / `CostumeSkirt` の3トグル）を MA Menu Installer で挿入しています。

### 1. アニメーションクリップを用意する

衣装のON/OFFはアニメーションクリップで制御します。COS_A のアニメーションクリップを参考に、衣装オブジェクトの `GameObject.IsActive` を ON/OFF するクリップを作成してください。

### 2. Animator Controller に組み込む

作成したクリップを Animator Controller のレイヤーに追加し、パラメーター（Bool）でON/OFFを切り替えるステートマシンを構成します。COS_A の構成をコピーして改変すると手早く済みます。

### 3. MA Menu Item と MA Menu Installer を追加する

衣装オブジェクトに `Add Component` で **MA Menu Item** と **MA Menu Installer** の両方を追加します。

**MA Menu Item の設定：**

| 項目 | 値 |
|------|-----|
| Name | ラベル名 |
| Icon | 任意のアイコン画像 |
| Type | Toggle |
| Parameter | Animator Controller のパラメーター名 |
| Value | 1 |

**MA Menu Installer の設定：**

| 項目 | 値 |
|------|-----|
| Install Target Menu | `3+1 OutfitMenu.asset` |

<TipsCard>COS_A の `SA_COS_A OutfitMenu.asset` を参考にアイコンとパラメーター名を設定すると、体裁を揃えやすいです。</TipsCard>

## 衣装固有の変化を制御する

衣装によってシェイプキーや Physbone の構成が変わる場合、アニメーションではなく MA のコンポーネントで動的に制御します。

### MA Shape Changer

衣装着用時に本体メッシュのシェイプキーを変化させたい場合に使います。衣装オブジェクトに `Add Component` → **MA Shape Changer** を追加し、対象のシェイプキーと値を設定してください。衣装のON/OFFに連動して自動で適用・解除されます。

COS_A では以下のシェイプキーを MA Shape Changer で制御しています。衣装を OFF にするとこれらの値は 0 に戻ります。これは MA Shape Changer の仕様です。

| シェイプキー | 衣装 ON | 衣装 OFF |
|-------------|---------|---------|
| `adj_bust_cloth1` | 100 | 0 |
| `adj_bust_inner_omitting` | 100 | 0 |
| `adj_shrink_hip(s)1` | 100 | 0 |
| `opt_hekochiku` | 10 | 0 |

<TipsCard>別の衣装でも同じシェイプキーを制御したい場合は、その衣装オブジェクトにも MA Shape Changer を追加して同じ値を設定してください。</TipsCard>

### MA Object Toggle

衣装着用時に特定のオブジェクト（Physbone コライダーなど）をON/OFFしたい場合に使います。衣装オブジェクトに `Add Component` → **MA Object Toggle** を追加し、対象オブジェクトと切り替えたい状態を設定してください。

## MA Parameters を合わせる

COS_A と同じパラメーター設定にする場合は、衣装オブジェクトに **MA Parameters** を追加して以下のように設定します。

| パラメーター名 | 型 | デフォルト値 | Saved |
|--------------|-----|------------|-------|
| `CostumeHead` | Bool | 0 | OFF |
| `CostumeBody` | Bool | 0 | OFF |
| `CostumeSkirt` | Bool | 0 | OFF |

<TipsCard>Saved を OFF にするとワールド移動時にトグルがリセットされます。常に状態を保持したい場合は Saved を ON にしてください。</TipsCard>

---

## MA Merge Animator を合わせる

衣装に独自のアニメーターがある場合は **MA Merge Animator** を追加して、COS_A と同じ設定にします。

| 設定項目 | 値 |
|----------|-----|
| Animator to Merge | 衣装用 Animator Controller |
| Layer Type | FX |
| Delete Attached Animator | ON |
| Path Mode | Relative |
| Match Avatar Write Defaults | ON |

---

## lilToon の設定を合わせる

プリファレンスメニューの **明度**（`Brightness` パラメーター）と **ライトのモノクロ化**（`Lighting(mono)` パラメーター）は、COS_A のアニメーターが各マテリアルの `_LightMinLimit` / `_MonochromeLighting` を直接操作することで機能しています。

新しく追加したメッシュのマテリアルはこのアニメーターの対象外になるため、**デフォルト値を本体と揃えておく**ことで、ラジアル未操作時の見た目を統一できます。

詳細は [シェーダー設定](/3plus1/settings/) を参照してください。

| プロパティ | 最小値（ラジアル 0） | 最大値（ラジアル 1） |
|-----------|------|------|
| `_LightMinLimit` | 0.05 | 1.0 |
| `_MonochromeLighting` | 0.5 | 1.0 |

<TipsCard>ラジアルで明度やモノクロを動かしたときに新規パーツだけ追従させたい場合は、COS_A のアニメーションクリップに新規メッシュのマテリアルスロットを追記する必要があります。</TipsCard>
