# 헤어·의상 추가（Modular Avatar）

Modular Avatar（MA）를 사용한 의상의 기본 설정（MA Merge Armature 등）은 [공식 튜토리얼](https://modular-avatar.nadena.dev/ja/docs/tutorials)을 참조하시기 바랍니다.

이 페이지에서는 MA로 의상이 작동하는 상태를 전제로, **3+1의 포맷에 맞추는** 절차를 설명합니다.

<WarningCard>
**여러 의상을 하나의 아바타에 묶는 것은 권장하지 않습니다.**
메뉴 전환으로 여러 의상을 하나의 아바타에 넣으면 모든 의상의 텍스처·메시가 항상 포함되므로, 아바타 다운로드 크기가 수백 MB에 달할 수 있습니다. 의상마다 별도의 아바타로 업로드하는 것을 검토해 주십시오.
</WarningCard>

<WarningCard>
**SA_COS_A를 먼저 삭제해 주십시오.**
새 의상을 추가하기 전에 Hierarchy에서 `SA_COS_A` 오브젝트를 삭제합니다. SA_COS_A를 남긴 채 다른 의상을 추가하면 클로젯 메뉴에 COS_A 토글이 계속 남아 파라미터와 외관이 의도치 않은 상태가 됩니다.
</WarningCard>

## 클로젯 메뉴에 추가하기

3+1의 클로젯은 `3+1 OutfitMenu.asset`이라는 서브메뉴에 각 의상이 토글을 삽입하는 구조로 되어 있습니다. COS_A는 이 서브메뉴에 `SA_COS_A OutfitMenu.asset`（`CostumeHead` / `CostumeBody` / `CostumeSkirt` 3개의 토글）을 MA Menu Installer로 삽입하고 있습니다.

### 1. 애니메이션 클립 준비하기

의상의 ON/OFF는 애니메이션 클립으로 제어합니다. COS_A의 애니메이션 클립을 참고로 의상 오브젝트의 `GameObject.IsActive`를 ON/OFF하는 클립을 작성해 주십시오.

### 2. Animator Controller에 조합하기

작성한 클립을 Animator Controller의 레이어에 추가하고, 파라미터（Bool）로 ON/OFF를 전환하는 스테이트 머신을 구성합니다. COS_A의 구성을 복사하여 수정하면 빠르게 완성할 수 있습니다.

### 3. MA Menu Item과 MA Menu Installer 추가하기

의상 오브젝트에 `Add Component`로 **MA Menu Item**과 **MA Menu Installer** 모두를 추가합니다.

**MA Menu Item 설정：**

| 항목 | 값 |
|------|-----|
| Name | 라벨 이름 |
| Icon | 임의의 아이콘 이미지 |
| Type | Toggle |
| Parameter | Animator Controller의 파라미터 이름 |
| Value | 1 |

**MA Menu Installer 설정：**

| 항목 | 값 |
|------|-----|
| Install Target Menu | `3+1 OutfitMenu.asset` |

<TipsCard>COS_A의 `SA_COS_A OutfitMenu.asset`을 참고하여 아이콘과 파라미터 이름을 설정하면 형식을 맞추기 쉽습니다.</TipsCard>

## 의상 고유의 변화 제어하기

의상에 따라 셰이프 키나 PhysBone의 구성이 달라지는 경우, 애니메이션이 아닌 MA 컴포넌트로 동적으로 제어합니다.

### MA Shape Changer

의상 착용 시 본체 메시의 셰이프 키를 변화시키고 싶을 때 사용합니다. 의상 오브젝트에 `Add Component` → **MA Shape Changer**를 추가하고 대상 셰이프 키와 값을 설정해 주십시오. 의상의 ON/OFF에 연동하여 자동으로 적용·해제됩니다.

COS_A에서는 다음 셰이프 키를 MA Shape Changer로 제어하고 있습니다. 의상을 OFF로 하면 이 값들은 0으로 돌아갑니다. 이는 MA Shape Changer의 사양입니다.

| 셰이프 키 | 의상 ON | 의상 OFF |
|-------------|---------|---------|
| `adj_bust_cloth1` | 100 | 0 |
| `adj_bust_inner_omitting` | 100 | 0 |
| `adj_shrink_hip(s)1` | 100 | 0 |
| `opt_hekochiku` | 10 | 0 |

<TipsCard>다른 의상에서도 같은 셰이프 키를 제어하고 싶은 경우, 해당 의상 오브젝트에도 MA Shape Changer를 추가하여 같은 값을 설정해 주십시오.</TipsCard>

### MA Object Toggle

의상 착용 시 특정 오브젝트（PhysBone 콜라이더 등）를 ON/OFF하고 싶을 때 사용합니다. 의상 오브젝트에 `Add Component` → **MA Object Toggle**을 추가하고 대상 오브젝트와 전환할 상태를 설정해 주십시오.

## MA Parameters 맞추기

COS_A와 동일한 파라미터 설정을 사용하려면 의상 오브젝트에 **MA Parameters**를 추가하고 아래와 같이 설정합니다.

| 파라미터 이름 | 타입 | 기본값 | Saved |
|--------------|-----|------------|-------|
| `CostumeHead` | Bool | 0 | OFF |
| `CostumeBody` | Bool | 0 | OFF |
| `CostumeSkirt` | Bool | 0 | OFF |

<TipsCard>Saved를 OFF로 설정하면 월드 이동 시 토글이 리셋됩니다. 항상 상태를 유지하려면 Saved를 ON으로 설정하십시오.</TipsCard>

## MA Merge Animator 맞추기

의상에 고유 애니메이터가 있는 경우 **MA Merge Animator**를 추가하고 COS_A와 동일한 설정으로 맞춥니다.

| 설정 항목 | 값 |
|----------|-----|
| Animator to Merge | 의상용 Animator Controller |
| Layer Type | FX |
| Delete Attached Animator | ON |
| Path Mode | Relative |
| Match Avatar Write Defaults | ON |

## lilToon 설정 맞추기

프리퍼런스 메뉴의 **밝기**（`Brightness` 파라미터）와 **라이트 모노크로화**（`Lighting(mono)` 파라미터）는 COS_A의 애니메이터가 각 머티리얼의 `_LightMinLimit` / `_MonochromeLighting`을 직접 조작함으로써 기능합니다.

새로 추가한 메시의 머티리얼은 이 애니메이터의 대상에서 제외되므로, **기본값을 본체와 맞춰 두면** 라디얼 미조작 시 외관을 통일할 수 있습니다.

자세한 내용은 [셰이더 설정](/3plus1/settings/)을 참조하십시오.

| 프로퍼티 | 최솟값（라디얼 0） | 최댓값（라디얼 1） |
|-----------|------|------|
| `_LightMinLimit` | 0.05 | 1.0 |
| `_MonochromeLighting` | 0.5 | 1.0 |

<TipsCard>라디얼로 밝기나 모노크로를 조작할 때 새 파츠만 따라오게 하려면 COS_A의 애니메이션 클립에 새 메시의 머티리얼 슬롯을 추가해야 합니다.</TipsCard>
